Задача:
Перечислить заголовки тест-кейсов для простой формы поиска(поле ввода с авто-подсказками + кнопка поиск) в порядке приоритета.

Приоритет 1

Группа 1: Ввод запроса в поле
1. Ввод валидного запроса (сущность присутствует в системе)
2. Ввод невалидного запроса (сущность отсутствует в системе)
3. Проверка лимита числа символов, вводимых в поле

Группа 2: Авто-подсказки
1. Появление авто-подсказки при вводе валидного запроса
2. Точность совпадения при различных запросах
3. Отсутствие авто-подсказки при вводе невалидного запроса
4. Применение авто-подсказки

Группа 3: Выполнение запроса
1. Выполнение запроса через кнопку поиска
2. Выполнение запроса через нажатие Enter

Группа 4: Адаптивность
1. Проверка UI в различных браузерах (основываясь на анализе наиболее популярных браузеров у ЦА)
2. Проверка UI при различных разрешениях экрана
3. Проверка UI на интуитивное понимание функционала пользователем

Приоритет 2

Группа 1: Ввод запроса в поле
1. Ввод пустого запроса
2. Ввод запроса, содержащего в себе html, sql и иные скрипты
3. Ввод запроса, содержащего в себе цифры и символы
4. (Опционально, если функция реализована) Ввод запроса через пареметр в URL
5. Ввод запроса через ctrl+v и иные функции вставки текста

Группа 2: Авто-подсказки
1. Пролистывание подсказок с клавиатуры
2. Появление подсказок при различных способах ввода (пункт 4 в прошлой группе и иные способы)
3. Появление/исчезновение подсказок при наведении/убирании фокуса с поля поиска

Группа 3: Выполнение запроса
1. Проверка консоли и сетевой вкладки в браузере при выполнении запроса

Группа 4: Адаптивность
1. Перемещение фокуса по элементам через Tab
2.(Опционально, если функция реализована) Мультиязычность
3. Проверка UI при различном масштабе
4. Проверка UI на мобильных версиях браузеров
5.(Опционально)Проверка HTML-разметки на наличие тегов, воспринимаемых программами для озвучки
